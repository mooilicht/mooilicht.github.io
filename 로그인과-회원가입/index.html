로그인과 회원가입

리액트 내비게이션

https://reactnavigation.org

리액트 내비게이션 라이브러리는 리액트 네이티브 애플리케이션의 내비게이션을 쉽고 간단하게 관리할 수 있도록 도와준다. 지원하는 내비게이션의 종류는 스택(stack), 탭(tab), 드로어(drawer)가 있다.

npm install --save @react-navigation/native

리액트 내비게이션은 각 기능별로 모듈이 분리되어 있어 사용하는 내비게이션 종류에 따라 개별적으로 추가 라이브러리를 설치해야 한다. 대부분의 내비게이션에서 반드시 필요한 종속성을 설치한다.

expo install
              react-native-gesture-handler
              react-native-reanimated
              react-native-screens
              react-native-safe-area-context
              @react-native-community/masked-view

스택 내비게이션

npm install @react-navigation/stack

스택 내비게이션은 가장 많이 사용하는 내비게이션으로 현재 화면 위에 다른 화면을 쌓으면서 화면이 이동한다.
화면 위에 새로운 화면을 쌓으면서 이동하기 때문에 이전 화면을 계속 유지한다.
가장 위에 있는 화면을 들어내면 이전 화면으로 돌아갈 수 있다.

화면 구성

createStackNavigator 함수를 이용해서 스택 내비게이션을 생성한다. 생성된 스택 내비게이션에는 화면을 구성하는 Screen 컴포넌트와 Screen 컴포넌트를 관리하는 Navigator 컴포넌트가 있다.

import { createStackNavigator } from "@react-navigation/stack";
import { Login, Signup } from "../screens";

const Stack = createStackNavigator();

const AuthStack = () => {
  return (
    <Stack.Navigator initialRouteName="Login">
      <Stack.Screen name="Login" component={Login} />
      <Stack.Screen name="Signup" component={Signup} />
    </Stack.Navigator>
  );
};

export default AuthStack;

스택 내비게이션에서 첫 번째 화면으로 나오는 화면은 Navigator 컴포넌트의 첫 번째 자식 Screen 컴포넌트다. initialRouteName 속성을 이용하면 첫 번째 화면을 지정할 수 있다.

화면 이동

const Login = ({ navigation }) => {
  return (
    <TouchableOpacity onPress={() => navigation.navigate("Signup")}>
      <WhiteText>회원이 아니신가요?</WhiteText>
    </TouchableOpacity>
  );
};

Screen 컴포넌트의 component로 지정된 컴포넌트는 화면으로 이용되고 navigation이 props로 전달된다.
navigation에는 다양한 기능이 있는데 그중 navigate 함수는 원하는 화면으로 이동하는 데 사용되는 함수다.

타이틀 수정하기

const AuthStack = () => {
  return (
    <Stack.Navigator initialRouteName="Login">
      <Stack.Screen
        name="Login"
        component={Login}
        options={{ headerShown: false }}
      />
      <Stack.Screen
        name="Signup"
        component={Signup}
        options={{
          headerTitle: "",
          headerBackTitleVisible: false,
          headerBackImage: ({ tintColor }) => {
            const style = {
              marginRight: 5,
              marginLeft: Platform.OS === "ios" ? 11 : 0,
            };
            return (
              <MaterialIcons
                name="arrow-back-ios"
                size={24}
                color={tintColor}
                style={style}
              />
            );
          },
        }}
      />
    </Stack.Navigator>
  );
};

  • headerShown : 헤더를 감출 수 있다. Navigator 컴포넌트의 screenOptions에 설정하면 전체 화면의 헤더가 보이지 않도록 설정할 수 있다.

  • headerTitle : 헤더 타이틀은 Screen 컴포넌트의 name 속성을 기본값으로 사용한다. 해당 속성을 사용하면 name 속성을 변경하지 않고 타이틀을 수정할 수 있다.

  • headerBackTitleVisible : 헤더 왼쪽에 뒤로 가기 버튼이 생성되는 것은 같지만 안드로이드는 버튼의 타이틀을 보여주지 않고, iOS는 버튼의 타이틀을 보여준다. 해당 속성을 사용하면 두 플랫폼의 버튼 타이틀 렌더링 여부를 동일하게 설정할 수 있다.

  • headerBackImage : 두 플랫폼이 동일한 이미지를 렌더링하도록 변경할 수 있다.

Hooks

useState

useState는 함수형 컴포넌트에서 상태를 관리한다.

const [state, setState] = useState(initialState);

useState 함수를 호출하면 상태를 관리하는 변수와 그 변수를 수정할 수 있는 세터 함수를 배열로 반환한다.
상태를 변경하려면 반드시 세터 함수를 이용해 값을 변경한다.

import React, { useState } from "react";

const App = () => {
  const [isReady, setIsReady] = useState(true);

  return isReady ? (
    <ThemeProvider></ThemeProvider>
  ) : (
    <AppLoading onFinish={() => setIsReady(true)} />
  );
};

useRef

특정 컴포넌트로 포커스를 설정하고 싶은 경우 useRef를 사용한다.

const ref = useRef(initialValue);

컴포넌트의 ref로 지정하면 생성된 변수에 값이 저장되는 것이 아니라 변수의 .current 프로퍼티(property)에 해당 값을 담는다. useRef의 내용이 변경돼도 다시 렌더링되지 않는다.

import React, { useRef } from "react";

const Login = ({ navigation }) => {
  const passwordRef = useRef(null);

  return (
    <Contents>
      <Input
        value={email}
        onSubmitEditing={() => passwordRef.current.focus()}
      />
      <Input ref={passwordRef} value={password} />
    </Contents>
  );
};

useEffect

useEffect는 컴포넌트가 렌더링될 때마다 원하는 작업이 실행되도록 설정할 수 있다.

useEffect(() => {}, []);

첫 번째 파라미터로 전달된 함수는 조건을 만족할 때마다 호출되고,
두 번째 파라미터로 전달되는 배열에 상태를 관리하는 변수를 전달한다.

import React, { useState, useEffect } from "react";

const Login = ({ navigation }) => {
  const [disabled, setDisabled] = useState(true);

  useEffect(() => {
    setDisabled(!(email && password && !errorMessage));
  }, [email, password, errorMessage]);
};

만약 두 번째 파라미터에 빈 배열을 전달하면 컴포넌트가 처음 렌더링될 때만 함수가 호출된다.

Context API

데이터를 전역적으로 관리하고 사용할 수 있도록 하는 기능이다. Context API를 이용하면 Context를 생성해 필요한 컴포넌트에서 데이터를 바로 받아올 수 있다.

const Context = createContext(defaultValue);

Context를 생성하는 createContext 함수는 파라미터에 생성되는 Context의 기본값을 지정할 수 있다.

import React, { useState, createContext } from "react";

const ProgressContext = createContext({
  inProgress: false,
  spinner: () => {},
});

const ProgressProvider = ({ children }) => {
  const [inProgress, setInProgress] = useState(false);
  const spinner = {
    start: () => setInProgress(true),
    stop: () => setInProgress(false),
  };
  const value = { inProgress, spinner };
  return (
    <ProgressContext.Provider value={value}>
      {children}
    </ProgressContext.Provider>
  );
};

export { ProgressContext, ProgressProvider };

Context 오브젝트는 입력된 기본값 외에 Consumer 컴포넌트와 Provider 컴포넌트를 갖고 있다.

  • Consumer : 상위 컴포넌트 중 가장 가까운 곳에 있는 Provider 컴포넌트가 전달하는 데이터를 이용한다. 만약 상위 컴포넌트 중 Provider 컴포넌트가 없다면 createContext 함수의 파라미터로 전달된 기본값을 사용한다. Consumer 컴포넌트의 자식은 반드시 리액트 컴포넌트를 반환하는 함수여야 하고, 이 함수는 Context의 현재값을 파라미터로 전달받아 데이터를 사용할 수 있다.

  • Provider : 하위 컴포넌트에 Context의 변화를 알리는 역할을 한다. Provider 컴포넌트는 value를 받아서 모든 하위 컴포넌트에 전달하고, 하위 컴포넌트는 Provider 컴포넌트의 value가 변경될 때마다 다시 렌더링된다. Provider 컴포넌트로부터 value를 전달받는 하위 컴포넌트의 수에는 제한이 없다.

useContext

useContext라는 Hook은 Consumer 컴포넌트의 자식 함수로 전달되던 값과 동일한 데이터를 반환하므로 Consumer 컴포넌트를 사용하지 않고 Context의 내용을 사용할 수 있게 해준다.

import React, { useContext } from "react";
import { ProgressContext } from "../contexts";

const Navigation = () => {
  const { inProgress } = useContext(ProgressContext);

  return <NavigationContainer>{inProgress && <Spinner />}</NavigationContainer>;
};

export default Navigation;

useContext를 사용하면 Consumer 컴포넌트 보다 사용법이 훨씬 간편하고 코드의 가독성도 좋아진다.

expo-image-picker

https://docs.expo.dev/versions/v38.0.0/sdk/imagepicker

기기의 사진이나 영상을 가져올 수 있도록 시스템 UI에 접근할 수 있는 기능을 제공한다.

expo-permissons도 같이 설치한다.

import React, { useEffect } from "react";
import { Platform, Alert } from "react-native";
import * as ImagePicker from "expo-image-picker";
import * as Permissions from "expo-permissions";

const Image = () => {
  useEffect(() => {
    (async () => {
      try {
        if (Platform.OS === "ios") {
          const { status } = await Permissions.askAsync(
            Permissions.CAMERA_ROLL
          );
          if (status !== "granted") {
            Alert.alert(
              "Photo Permission",
              "Please tun on the camera roll permissions."
            );
          }
        }
      } catch (e) {
        Alert.alert("Photo Permission Error", e.message);
      }
    })();
  }, []);

  const _handleEditButton = async () => {
    try {
      const result = await ImagePicker.launchImageLibraryAsync({
        mediaType: ImagePicker.MediaTypeOptions.Images,
        allowsEditing: true,
        aspect: [1, 1],
        quality: 1,
      });

      if (!result.cancelled) {
        onChangeImage(result.uri);
      }
    } catch (e) {
      Alert.alert("Photo Error", e.message);
    }
  };
};

안드로이드는 특별한 설정 없이 사진에 접근할 수 있지만,
iOS는 사진첩에 접근하기 위해 사용자에게 권한을 요청하는 과정이 필요하다.

사진 변경 버튼을 클릭하면 호출되는 함수에서 기기의 사진에 접근하기 위해 호출되는 라이브러리 함수는 다음과 같은 값들을 포함한 객체를 파라미터로 전달받는다.

  • mediaTypes : 조회하는 자료의 타입
  • allowsEditing : 이미지 선택 후 편집 단계 진행 여부
  • aspect : 안드로이드 전용 옵션으로 이미지 편집 시 사각형의 비율([x, y])
  • quality : 0~1 사이의 값을 받으며 압축 품질을 의미(1: 최대 품질)

조회하는 자료는 사진으로 설정하고, 크기가 다양한 사진을 편집하기 위해 편집 단계를 진행하도록 설정한다.
iOS는 정사각형이므로 안드로이드에서도 1:1 비율로 편집되도록 설정했고 품질은 최대로 설정한다.

기기의 사진에 접근하는 함수는 결과를 반환하는데, 반환된 결과의 cancelled 값을 통해 선택 여부를 확인할 수 있다.
만약 사용자가 사진을 선택했다면 반환된 결과의 uri를 통해 선택된 사진의 주소를 알 수 있다.

결과에 따라 선택된 사진의 uri를 props로 전달된 onChangeImage 함수에 파라미터로 전달하며 호출하도록 작성한다.

const Signup = () => {
  return (
    <KeyboardAwareScrollView extraScrollHeight={20}>
      <Container>
        <Image onChangeImage={(url) => setPhotoUrl(url)} />
      </Container>
    </KeyboardAwareScrollView>
  );
};

파이어베이스

https://console.firebase.google.com

파이어베이스(Firebase)는 인증(Authentication), 데이터베이스(Database) 등의 다양한 기능을 제공하는 개발 플랫폼이다. 파이어베이스가 제공하는 기능을 이용하면 별도의 서버 구축 없이 파이어베이스를 이용해 개발이 가능하다.

외부로 노출되면 안되는 값들은 firebase.json 파일을 생성하여 관리한다. .gitignore에 firebase.json을 추가한다.

  • 인증 : 다양한 인증 방법을 제공한다. 이메일과 비밀번호만 사용해서 만들었다.
  • 데이터베이스 : 서울인 asia-northeast3을 선택했다.
  • 스토리지 : 서버 코드 없이 사용자의 사진, 동영상 등을 지정할 수 있는 기능을 쉽게 개발할 수 있도록 기능을 제공한다.

회원가입

import { getAuth, createUserWithEmailAndPassword } from "firebase/auth";

const auth = getAuth();

export const signup = async ({ email, password }) => {
  const { user } = await createUserWithEmailAndPassword(auth, email, password);
  return user;
};

이메일과 비밀번호를 이용해서 사용자를 생성하는 함수는 createUserWithEmailAndPassword다.

import { getStorage, ref, uploadBytes, getDownloadURL } from "firebase/storage";

const storage = getStorage();

const uploadImage = async (uri) => {
  const blob = await new Promise((resolve, reject) => {
    const xhr = new XMLHttpRequest();
    xhr.onload = function () {
      resolve(xhr.response);
    };
    xhr.onerror = function (e) {
      reject(new TypeError("Network request failed"));
    };
    xhr.responseType = "blob";
    xhr.open("GET", uri, true);
    xhr.send(null);
  });

  const user = auth.currentUser;
  const storageRef = ref(storage, `/profile/${user.uid}/photo.png`);
  await uploadBytes(storageRef, blob, {
    contentType: "image/png",
  });

  return await getDownloadURL(storageRef);
};

export const signup = async ({ email, password, name, photoUrl }) => {
  const storageUrl = photoUrl.startsWith("https")
    ? photoUrl
    : await uploadImage(photoUrl);
  await user.updateProfile({
    displayName: name,
    photoURL: storageUrl,
  });
};

회원가입시 사용자 정보에 스토리지를 이용하여 사용자의 사진 주소와 이름을 추가한다.

const Signup = () => {
  const _handleSignupButtonPress = async () => {
    try {
      const user = await signup({ email, password, name, photoUrl });
    } catch (e) {}
  };
};

스토리지 보안 규칙을 다음과 같이 수정한다.

service firebase.storage {
  match /b/{bucket}/o {
    match /profile/{userId}/photo.png {
        allow read;
      allow write: if request.auth.uid == userId;
    }
  }
}
