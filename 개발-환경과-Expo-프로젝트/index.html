개발 환경과 Expo 프로젝트

개발 환경

노드랑 에디터는 앞서 블로그를 만들면서 설치했다.

파이썬

https://www.python.org/downloads

리액트 네이티브는 빌드할 때 파이썬 2 버전이 필요하다.

JDK

https://www.oracle.com/java/technologies/downloads

안드로이드 개발 언어는 자바. JDK(Java Development Kit)가 필요하다.

환경 변수 - 시스템 변수

  • JAVA_HOME 변수 추가 : 값은 C:\Program Files\Java\jdk1.8.0_331
  • Path 변수에 %JAVA_HOME%\bin 추가

명령 프롬프트 창에서 확인한다.

java -version
javac -version

안드로이드 스튜디오

https://developer.android.com/studio

안드로이드 스튜디오는 안드로이드를 개발하기 위한 공식 IDE(Integrated Development Environment)다.
iOS의 Xcode와 같은 역할이다.

  • Install Type : Custom

나머지 설정은 기본 설정으로 진행한다.

SDK Manager

  1. SDK Platforms 탭 오른쪽 하단에 있는 Show Package Details를 클릭한다.
  2. API Level은 설치할 에뮬레이터에 맞췄다. Android SDK Platform과 Intel x86 Atom_64 System Image 또는 Google APIs Intel x86 Atom System Image를 설치한다.

환경 변수

  • 사용자 변수에 ANDROID_HOME 변수 추가 : %LOCALAPPDATA%\Android\Sdk
  • 시스템 변수 Path 변수에 %LOCALAPPDATA%\Android\Sdk\platform-tools

명령 프롬프트 창에서 확인한다.

adb --version

에뮬레이터

  1. Virtual Device Manager를 클릭한다.
  2. Create Device를 클릭하여 안드로이드 가상 기기를 만든다.
  3. 이미지 선택 화면에서 x86 최신 버전으로 선택한 후 진행한다.
  4. 녹색 시작 버튼을 클릭해서 에뮬레이터를 실행한다.

Expo

https://expo.io

Expo는 리액트 네이티브를 처음 시작하는 사람도 프로젝트를 쉽게 배포 및 관리할 수 있도록 다양한 기능을 제공하는 툴이다. Xcode, 안드로이드 스튜디오 없이도 테스트를 진행하며 개발할 수 있다.

단점은 Expo에서 제공하는 API만 사용할 수 있으며 필요한 기능이 제공되지 않을 경우 네이티브 모듈을 추가로 만들어서 사용하는 것이 불가능하다. 리액트 네이티브 CLI를 이용했을 때 빌드 파일의 크기가 크다는 것도 단점이다.

Expo를 사용하려면 회원가입을 해야 한다.

Expo 설치

해당 명령어로 expo를 설치한다.

npm install --global expo-cli

프로젝트 생성

해당 명령어로 프로젝트를 생성한다.

expo init <프로젝트명>

선택지가 제공되는데 blank를 선택했다.

프로젝트 실행

프로젝트를 실행하기 전 Expo에 로그인부터 한다.

expo login

해당 명령어로 프로젝트를 실행한다.

npm start

안드로이드 에뮬레이터에서 화면을 확인하려면

a

iOS는 아이폰을 가지고 있어 애플 앱 스토어에서 Expo 애플리케이션을 설치해서 사용했다.
